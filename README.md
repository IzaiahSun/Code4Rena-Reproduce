# Code4Rena Vulnerability Reproduce

This is a repo that contains reproduced proof-of-concept (PoC) of bugs reported by Code4Rena auditors.

Most of the bugs are reproduced by AI and some are checked by human.

## Project Structure
```
|- README.md
|- projects/
   |- <year-month-project>

```

## Additional files

Under each project, there are **three** additional files.

- `setup_env.sh` is a script to help install the dependency.
- `run_pocs.sh` is a script to run all the PoCs.
- `poc_manifest.json` is a json file to list all the generated PoCs.

Both `setup_env.sh` and `run_pocs.sh` should be run with the project directory as the current working directory, e.g.:

```bash
cd projects/2025-02-recall
./setup_env.sh
./run_pocs.sh
```

When building the environment, several package managers/commands are considered already existing on the machine (in `PATH`):
```
npm
pnpm
npx
forge
```

## PoC files

For each PoC file, in the beginning, there are descriptions about the root cause of the vulnerability, invocation chains and other informations.
These information are generated by AI and we do not gurantee the correctness.

## What is going on.

We are collecting **Solidity** projects and generate PoCs for **High Risk** findings.
We plan to expand to other risk levels and other languages when Solidity High is done.
